# Passive Safety Software Change Log

## 被动安全控制器软件变更记录 Skill

### 用途
从软件/基线管理后台截图中提取控制器软件变更信息，并持续记录到同一个飞书电子表格中。

### 适用场景
- 发送软件/电检基线/MRD基线/OTA基线后台截图，要求记录控制器软件变更
- 需要长期沉淀被动安全控制器（如 APSB、BELM 等）的版本信息
- 需要结构化归档：控制器名称、软件ID、软件版本、硬件信息、供应商等

### 提取字段
| 字段 | 说明 |
|---|---|
| 控制器名称 | ECU名称，如 APSB、BELM |
| 软件ID | 如 0101、0005 |
| 软件版本 | 16进制计数，如 0009 → 000A |
| 车型项目 | 如 W10国内、W10海外、W05 |
| 项目阶段 | 如 P2-2、SOP、EP2、PP4、OTA9.0.1 |
| 基线类型 | 电检 / MRD / OTA |
| 供应商ID | 如 6DS、6JC |
| 硬件零件号 | 如 LAU-56020011、LAE-56020109 |
| 硬件版本 | 16进制计数 |
| BT版本 | 16进制计数 |
| 原始标题 | 页面完整标题，用于追溯 |
| 记录时间 | 录入时间 |
| 原始备注 | 补充信息 |

### 识别规则
1. **只关注标题和ECU对应行**，忽略统计卡片、创建人等辅助信息
2. **标题拆解**：将页面标题拆为 车型项目、项目阶段、基线类型
3. **多车型拆分**：若标题包含多个车型（如国内+海外），自动拆成多行记录
4. **业务字段优先从ECU行读取**，不参考页面顶部辅助信息

### 重复检测
写入前自动按以下组合查重：
- 控制器名称 + 软件ID + 软件版本 + 硬件零件号 + 车型项目

若已存在，会提醒你而不重复写入。

### 排序规则
1. 控制器名称（升序）
2. 软件ID（升序）
3. 软件版本（高版本在前，16进制排序）

### 使用示例
```
发一张基线后台截图，附带说明："记录被动安全控制器信息skill"
```

Skill 会自动：
1. 识别标题 → 拆解车型项目/项目阶段/基线类型
2. 定位 ECU名称 + 软件ID 所在行
3. 提取该行所有软件信息
4. 查重 → 写入飞书表格

### 注意事项
- 软件版本、硬件版本、BT版本为16进制计数（0009 → 000A → 000B）
- 截图分辨率不足时，无法识别的字段会标记"待确认"
- 多车型标题（如"W10国内P2-2&P3电检基线及海外PPV3&PPV4电检基线"）会自动拆成两行
